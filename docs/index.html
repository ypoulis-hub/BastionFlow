Why BastionFlow exists

Microsoft's official Windows App targets AVD end-users on assigned desktops. It does not handle the admin workflow of "connect to any VM in my tenant via Azure Bastion, regardless of AVD configuration, with Entra ID auth that survives hostname mismatches and stale device records."

If you've ever hit any of these, BastionFlow fixes them automatically:

  • AADSTS293004: The target-device identifier in the request <vmname> was not found in the tenant
  • This RDP File is corrupted. The remote connection cannot be started.
  • Your credentials did not work on Workplace-joined PCs where AAD-RDP needs a fresh PRT
  • The Bastion native client (az network bastion rdp) auto-launching the wrong RDP client

Build from source

# Prerequisites (build-time only)
winget install Microsoft.DotNet.SDK.8 JRSoftware.InnoSetup `
  --accept-package-agreements --accept-source-agreements

git clone https://github.com/ypoulis-hub/BastionFlow
cd BastionFlow
dotnet build
dotnet test
dotnet run --project src/BastionFlow.App

# Produce a Setup.exe under dist/
pwsh installer\build-installer.ps1
